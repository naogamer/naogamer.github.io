Configurando o computador - Como criar jogos de um jeito fácil [1]

2020-02-06 :: Albrigs
#desenvolvimento  #tutoriais  #primeiro-jogo 

Esse post é parte de uma série onde serão ensinadas bases da criação de jogos, não iremos a um nível muito profundo, pois para isso há cursos muito bem fundamentados para isso. Esta série é apenas uma introdução.

Esta não é a primeira postagem, para ver a lista clique aqui.

Por mais que este seja um tutorial para desenvolvimento ele é voltado para iniciantes, portanto evitaremos utilizar terminais e recursos de linha de comando. Eles por alguma razão são bem mais intimidadores para iniciantes do que deveriam e por isso não vamos evita-los. Isso será excessão apenas para a parte voltada para usuários Linux que, embora não seja obrigatório, é o jeito mais rápido de fazer as coisas.

Requisitos de máquina

Roda até no pc mais batata dos batatas.

Um detalhe muito bom sobre as tecnologias que usaremos é que elas rodam até em um Chromebook de 2GB de RAM e estou utilizando um para garantir isto. Diferente de jogos 3D os 2D não exigem, ou pelo menos não deveriam exigir, muito hardware para ser programados. Se lembrarmos bem a geração 16 bits que foi o berço e terreno de guerra entre franquias incríveis como Street Fighter, Super Mário e Sonic surgiu em uma época onde no máximo as memórias RAM chegavam a 128MB.

Claro que não usaremos exatamente as mesmas ferramentas, essa informação está aí para se ter uma noção de que hardware difícilmente será um problema. Se você usa Linux, IOs ou Windows não se preocupe, Python é feito para rodar em todos os computadores possíveis. Não dou garantias quanto a programar em um dispositivo Android, pois pode não ser algo ergonômico nem ter ferramentas muito boas. Caso tente e consiga sinta-se uma pessoa vitoriosa e que vai longe nessa vida de TI.

Instalando o Python

Provavelmente a única forma de seu computador não vir com Python já instalado é no caso de ele utilizar alguma versão de Windows. Para isso basta apenas entrar em python.org/downloads/ e baixar a ultima versão. O processo de instalar é o mesmo que para qualquer outro programa: duplo clique no instalador, próximo, próximo, próximo até acabar a instalação.

Atenção só para ter certeza que ele estará sendo adicionado ao PATH da sua máquina. Se não estiver você pode ter problemas.

Estaremos usando a versão 3 do Python, pois, como estamos em 2020, a versão 2 perdeu o suporte e algumas coisas que iremos usar podem não se dar bem com isso no futuro. O uso da versão 3 também é para que seja possível usar certos recursos indisponíveis na 2.

Se você usa Linux só precisa executar essas linhas no seu terminal:

# caso sua distrubuição não use apt verifique qual ela usa (provavelmente é yum)
sudo apt install python3
sudo apt install python3-pip

Instalando o Pycharm

Caso você deseje escrever seus códigos em um bloco de notas é totalmente possível e por incrível que pareça tem gente que ainda faz isso. Vamos usar algo que dá um pouco mais de assistência e agilidade, uma IDE, que é basicamente um editor de texto cheio de ferramentas que facilitam muito nossa vida.

Adianto que não é o que utilizo normalmente para programar, mas é o que eu recomendo para iniciantes.

A IDE que utilizaremos é a Pycharm em sua versão Community que, além de gratuita, tem tudo que precisaremos para escrever bem nosso código sem correr o risco de ter qualquer problema em nosso computador. Futuramente irei explicar como e quais problemas podemos causar em nossos computadores, só tenha consciência que estamos pegando as soluções todas de uma vez só.

Para baixar o Pycharm na versão que precisamos basta acessar jetbrains.com/pycharm/download/ e a instalação é nossa velha conhecida de next next next next…

Caso você seja um usuário Linux a forma mais estavel que eu encontrei de instalar o pycharm sem ser pelo arquivo tar.gz disponibilizado é pelo flatpak. Pode parecer ruim ter que reiniciar seu computador, mas garanto que é uma ferramenta muito boa para o que não está em seu gerenciador de pacotes principal. Meus testes com snap não deram bons resultados, tendo uma lentidão bem chata na abertura e execução de programas. O procedimento de instalação no seu terminal é basicamente:

sudo apt install flatpak

flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.fla$

Neste momento reinicie o computador.

Para instalar o pycharm é preciso usar o seguinte comando:

flatpak install flathub com.jetbrains.PyCharm-Community

Opcionais

Algumas ferramentas extra vão nos ajudar na parte de planejamento e de representação do que queremos e algumas podem ser substituidas por papel e caneta. A preferência pelo digital é que teremos que reescrever e editar algumas vezes as coisas durante o próprio processo de fazer o jogo. A primeira é um editor de texto que pode ser o Word, Google Docs, LibreOffice Writer ou até o Pycharm.

DIA - Diagramas

Pode não ser um programa bonito, mas é eficiente organizar nossas informações assim.

Dia é uma ferramenta muito boa para desenhar diagramas e esquemas. Eles são muito úteis para tornar visível as relações e características do que estamos planejando. Sua instalação não é diferente de qualquer outro programa no Windows: acesse dia-installer.de/ e siga os mesmos passos de sempre.

O comando de instação para usuários Linux fica:

sudo apt install dia

Alguns sites para ficar nos favoritos:

Algumas ferramentas que riemos usar serão retiradas de sites e serão elas:

  • Colour Lovers para nos dar algumas ideias de quais palhetas de cores usar;
  • Hailpixel que é um gerador de palhetas de cores;
  • Itch.io será uma fonte de assets que podemos usar no futuro caso precisemos de algum desenho. Ela é um grande marketplace para jogos indie e recursos para construir estes;
  • Webaim Contrast Checker que, por mais que seja mais voltado para texto e fontes, ajuda a nos dar noção se estamos usando cores boas ou vamos ter algum problema no meio do caminho;
  • Documentação do Python documentações são como os manuais de instruções da programação. Quando bem feitas elas te ensinam tudo o que você precisar, sendo sempre bom checar;
  • Documentação da Pygame por favor, não se intimide com o site que parece ter saido dos anos 90, ele é atual sim e embora precise de um tapa no visual é incrivelmente útil.

Essas são as configurações necessárias para que nos possamos começar. No próximo post já estaremos nos aventurando pelo planejamento do primeiro jogo, uma etapa fundamental para qualquer processo de desenvolvimento que não queira morrer na praia ou nunca chegar a lugar nenhum.
